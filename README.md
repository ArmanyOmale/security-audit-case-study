# Security Audit Case Study — Botium Toys (Fictional Organization)

## Project Overview

This project is a security audit conducted on a fictional company, Botium Toys, as part of my cybersecurity training. The objective was to assess the organization’s security posture, identify risks, and evaluate existing controls.

## Organizational Context

Botium Toys is a fictional retail company with online operations. The audit examined security practices across technical, administrative, and physical domains.

## Scope of Audit

The assessment focused on:

* Access control policies
* Network security controls
* Data protection measures
* Physical security safeguards
* Logging and monitoring practices
* Disaster recovery preparedness

## Risk Assessment Summary

Key risks identified included:

* Lack of encryption for sensitive data
* Weak password policies
* Absence of intrusion detection systems
* Inadequate disaster recovery planning
* Limited security monitoring

## Compliance & Controls Evaluation

Security controls were evaluated against industry standards and compliance frameworks including:

* PCI DSS
* GDPR
* SOC 1 / SOC 2

## Key Recommendations

* Implement stronger authentication controls
* Deploy intrusion detection and monitoring systems
* Enforce encryption for sensitive data
* Establish formal incident response procedures
* Improve backup and recovery processes

## Skills Demonstrated

* Security risk assessment
* Governance and compliance awareness
* Analytical thinking
* Security documentation
* Understanding of organizational security

## Purpose of Project

This project demonstrates foundational capabilities relevant to a SOC Analyst role, including risk analysis, investigation, and governance awareness.
